# Telegram Browser

Безопасный браузер для Telegram Web с SSH-туннелем через VPS.

## Описание

Приложение открывает Telegram Web в отдельном окне без элементов управления браузера. Весь трафик проходит через зашифрованный SSH-туннель на ваш VPS, что позволяет обходить блокировки и сохранять конфиденциальность.

## Возможности

- Чистый интерфейс без адресной строки и лишних кнопок
- SSH SOCKS5 туннель для защищённого соединения
- Настройка любого веб-приложения (не только Telegram)
- Сохранение параметров подключения
- Автоматическое переподключение
- Кроссплатформенность: Windows и macOS

## Устройство

Приложение состоит из двух процессов:

- Основной процесс (Node.js) управляет SSH-туннелем и окнами
- Процесс рендеринга (Chromium) отображает веб-приложение

Связь между процессами организована через IPC. Chromium изолирован и не имеет доступа к системе напрямую.

## Требования

- Node.js 18 или новее
- VPS с SSH-доступом и включённым TCP-форвардингом

## Установка и запуск

```bash
# Клонирование репозитория
git clone https://github.com/ваш-логин/tg-browser.git
cd tg-browser

# Установка зависимостей
npm install

# Запуск в режиме разработки
npm start
```

## Сборка

```bash
# Установщик для Windows
npm run build-win

# DMG для macOS
npm run build-mac

# Обе платформы
npm run build-all
```

Результат сборки находится в папке `dist`.

## Настройка VPS

На сервере должен быть разрешён TCP-форвардинг:

```bash
# В файле /etc/ssh/sshd_config
AllowTcpForwarding yes
PermitRootLogin yes

# Перезапуск SSH
systemctl restart sshd
```

## Использование

1. При первом запуске откроется окно настройки подключения
2. Укажите адрес приложения, данные VPS и пароль
3. Нажмите "Подключиться"
4. Приложение запустит туннель и откроет веб-интерфейс
5. При последующих запусках подключение произойдёт автоматически

Для сброса сохранённых настроек удалите файл `stored-config.json` в папке пользовательских данных приложения.

## Структура проекта

```
tg-browser/
├── assets/              # Иконки приложения
├── src/
│   ├── main.js          # Точка входа
│   ├── config/          # Управление конфигурацией

```

## Безопасность

- Пароль VPS не хранится в коде приложения
- Данные сохраняются в зашифрованном виде в пользовательской директории
- Chromium работает в изолированном режиме без доступа к Node.js
- Рекомендуется использовать отдельного пользователя на VPS вместо root
